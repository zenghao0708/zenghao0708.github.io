# 文章评分机制

## 目标

- 公开发布的文章，目标分数应 `>= 9.0`
- `8.5 - 8.9` 可以继续打磨后再发
- `< 8.5` 不建议直接发布；若与已有文章重合明显，优先重定位或下线

## 六个评分维度

总分 `10.0`，按以下维度打分：

1. 选题定位清晰度：`2.0`
2. 信息密度与结构推进：`2.0`
3. 实际指导意义：`2.5`
4. 论证证据与例子：`1.5`
5. 配图与可视化辅助：`1.0`
6. 与现有文章的差异化：`1.0`

## 评分标准

### 1. 选题定位清晰度：2.0

- `2.0` 读者在前 20 秒内就知道“这篇解决什么问题、适合谁看、和系列其他文章有什么区别”
- `1.0` 有主题，但和已有文章的边界不够清楚
- `0` 标题、导语和正文定位打架

### 2. 信息密度与结构推进：2.0

- `2.0` 结构层层推进，不重复，不空转
- `1.0` 有不少正确信息，但章节之间重复明显
- `0` 主要靠换说法重复同一个判断

### 3. 实际指导意义：2.5

- `2.5` 读者读完可以直接知道“应该怎么做、先做什么、为什么”
- `1.5` 有启发，但仍停留在观点层
- `0` 只有判断，没有可执行建议

### 4. 论证证据与例子：1.5

- `1.5` 有源码、案例、流程、模块、日志、截图等支撑
- `0.5` 主要靠作者判断，证据偏少
- `0` 几乎没有证据

### 5. 配图与可视化辅助：1.0

- `1.0` 图片明显帮助理解，而不是装饰
- `0.5` 有图，但和正文结合弱
- `0` 没图且本应有图

### 6. 差异化：1.0

- `1.0` 与站内已有文章边界清楚，可以同时保留
- `0.5` 有一定区别，但主题高度重叠
- `0` 可被另一篇更完整的文章直接替代

## 一票否决项

出现以下任意一项，文章最高只能评到 `8.5`：

- 没有回答“这篇文章适合谁看”
- 没有回答“读者读完能拿走什么”
- 和站内已有文章高重复，但没有明确重定位
- 全文只有观点，没有源码、案例、流程或图示支撑
- 需要图但没有图

## 发布前检查

- 是否有 `title / description / abbrlink / tags / categories`
- 是否有 `<!-- more -->`
- 是否在开头明确文章定位
- 是否至少有一段“可执行建议 / 最值得抄的作业 / 操作建议”
- 是否有必要的结构图、流程图、知识卡
- 是否给出相关阅读，避免系列文章互相抢位
- 默认发布入口 `node publish/publish-all.js` 会在发布前自动执行质量检查
- 默认门槛：`9.0`
- 可覆盖：`--quality-min <n>`
- 临时跳过：`--skip-quality-gate`
- 质量报告输出到：`publish/output/article-quality/`

## Claude Code 系列的定位建议

- `claude-code-source-why-better`
  - 角色：快速判断版 / 传播入口
- `claude-code-source-deep-dive`
  - 角色：结构图导读版
- `claude-code-source-analysis`
  - 角色：主稿 / 权威版

如果一篇新文章无法在这三种角色之外给出新的边界，原则上不应新增第四篇平行文章。
