  • Unable to read plist data from `”/Users/hoare/Git_Repo/Misfit-BLE-iOS-SDK/Misfit-BLE-iOS-SDK/Misfit-BLE-iOS-SDK.xcodeproj/project.pbxproj”‘: Unexpected character / at line 1

    • 出现这种情况的原因是，cocoapods的版本不兼容。

      • 方案一：清除 cocoapods 的本地缓存：

        cd ~/.cocoapods/repos rm -rf master pod setup

      • 方案二：安装特定版本的 cocoapods：

        卸载当前版本： sudo gem uninstall cocoapods

        安装0.32.1版本： sudo gem install cocoapods -v 0.32.1

  • error: The sandbox is not in sync with the Podfile.lock. Run ‘pod install’ or update your CocoaPods installation.

    • 这个错误一般会显示下面三行：
      • diff: /../Podfile.lock: No such file or directory
      • diff: /Manifest.lock: No such file or directory
      • error: The sandbox is not in sync with the Podfile.lock. Run ‘pod install’ or update your CocoaPods installation.

    • StackOverflow上的回答:

      • 删除xcworkspace文件，然后运行pod install

  • 运行“pod install”速度慢

    • 运行”pod install”命令对从github上面把所有的pod specs repo信息重新下载到本地，所以速度很慢，解决方法：

      • 先运行“pod setup”来下载specs文件，然后运行pod install

      • 或者运行“pod install –no-repo-update”

      • 无论什么时候出现问题，可以通过开启调试模式( —verbose)来查看详细log

  • 共享的Scheme:

    • 共享的Scheme与属于用户的Scheme有下面的异同点:

      • 都在xcodeproj这个目录里

      • 但是属于用户的Scheme: 存放在xcuserdata这个目录中,包含了每一个用户的Xcode设置项，每个用户都会一个属于自己的scheme（用户名.scheme）

      • 共享的Scheme: 存放在xcshareddata目录中,一个项目里只能有一份共享的scheme，每个人都可以更改和使用。
