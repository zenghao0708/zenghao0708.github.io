MIGRATION CHECKLIST

我最认可的迁移原则

这次迁移真正帮我避免翻车的，不是某个具体工具，而是这套“先拆结构、并行接管、最后切流”的方法。

1

先量化问题，再动配置

先看 shell 启动耗时、PATH 和 which，而不是凭体感判断“好像变快了”。

2

不要只验版本号，要验所有权

which node / mise which node 比单看 node -v 更关键。

3

旧链路要真正退出

迁移完成后继续清掉 .nvm、.rbenv 残留和重复加载，不然问题会在后面回潮。

4

先准备回滚，再谈切换

主力开发机迁移最怕的不是改错，而是改错以后没法快速退回稳定状态。
